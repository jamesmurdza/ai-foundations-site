# Hacker House Admin

Internal admin dashboard for the AI Foundations hacker house. Reads applications from the same Neon Postgres `hh_applications` table that powers the public application form (the form lives in [`ai-foundations-site`](https://github.com/jamesmurdza/ai-foundations-site)).

## What it does

- **List view** — every application sorted by submission/update time, with filters by admin decision (pending / accepted / waitlist / rejected) and flow status (submitted / in progress).
- **Detail view** — every applicant's 16 static answers, 5 dynamic follow-ups, "why" + project text, and links, with a sidebar to set the admin decision and write internal notes.
- **CSV export** — one click on the list page; full flat-table dump of every applicant + every answer.
- **HTTP basic auth** — the whole site is gated by basic auth via `src/middleware.ts`, configured by the `ADMIN_USERS` env var.

## Run locally

```
npm install
cp .env.example .env.local                  # then fill in real values
node scripts/db/migrate.mjs                 # admin_status + admin_notes columns
node scripts/db/migrate-admins.mjs          # hh_admins table (seeds from ADMIN_USERS once)
node scripts/db/migrate-comments.mjs        # hh_comments + hh_visits tables
node scripts/db/migrate-notifications.mjs   # notification_sent_at column
npm run dev                                 # http://localhost:3000
```

`.env.local` needs (see `.env.example` for the full list):

```
DATABASE_URL=postgresql://…
ADMIN_USERS=burhan:somepass,james:otherpass   # seed only

# Applicant thank-you email
SMTP_HOST=smtp.improvmx.com
SMTP_PORT=587
SMTP_USER=summerschool@aifoundations.school
SMTP_PASS=…
SMTP_FROM="AI Foundations <summerschool@aifoundations.school>"

# Cron backstop secret
CRON_SECRET=…
```

## Applicant emails

Every Summer School application submission gets a plain-text thank-you from `summerschool@aifoundations.school` to the applicant only (no CC). The instant send happens in the form repo (`ai-foundations-site`) inside the submit handler. This dashboard runs a Vercel Cron every 15 minutes against `/api/cron/notify-submissions` as a backstop — it picks up any submission whose `notification_sent_at` is still `NULL` and sends. Both paths use the same atomic claim (`UPDATE … WHERE notification_sent_at IS NULL RETURNING id`) so duplicates are not possible.

## Design

UI built against `DESIGN.md` (Apple-inspired, generated by `npx getdesign@latest add apple`):

- Single Action Blue (`#0066cc`) on every interactive element
- 17px body, near-black ink (`#1d1d1f`), parchment / pearl surfaces, hairline borders
- Pill-shaped primary buttons, `rounded-lg` (18px) for utility cards, no decorative shadows

When changing UI, read `DESIGN.md` first and reference its tokens.
